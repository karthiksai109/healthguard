## HOW TO GET ACCESS KEYS ON LIVE DEPLOYMENT
================================================================================
Since keys are randomly generated per deployment, check the server startup logs:

  - Patient keys: Look for "demo_patient_created" log entries
  - Doctor keys:  Look for "demo_doctor_created" log entries

Or use the API:
  - GET /patients           -> list all patient IDs
  - GET /doctors            -> list all doctors with details
  - Check Akash logs tab for startup output with all keys

## DEMO FEATURES TO SHOWCASE
================================================================================
1. PATIENT LOGIN:  Use any patient access key to log in
2. AI CHAT:        Streaming responses (tokens appear word-by-word)
3. PHOTO ANALYSIS: Upload wound/burn/rash photos for AI triage
4. VOICE INPUT:    Record voice for AI SOAP note generation
5. VITALS:         Add BP, heart rate, glucose, temperature, O2, pain
6. CHARTS:         Real-time vitals charts with trend visualization
7. AI REPORTS:     Generate doctor reports (click "AI Report" button)
8. AUDIO BRIEFING: AI-generated spoken patient briefing (TTS)
9. DOCTOR LOGIN:   Use doctor access key (starts with DR)
10. CONSULTATIONS: Patients can request doctor consultations
11. ENCRYPTION:    All sensitive data encrypted with AES-256-GCM
12. AUDIT LOG:     Immutable audit trail of all actions

## TECH STACK
================================================================================
- Backend:    FastAPI + Python on Akash Decentralized Cloud
- AI:         Venice AI (llama-3.3-70b) for chat, analysis, reports
- Vision:     Venice Vision API for medical image analysis
- Speech:     Venice STT (whisper) + TTS (kokoro)
- Encryption: AES-256-GCM (PBKDF2 key derivation)
- Database:   SQLite with WAL mode + encrypted fields
- Frontend:   Vanilla JS with streaming SSE for real-time chat
- Docker:     karthiksai109/healthguard:v2.4
